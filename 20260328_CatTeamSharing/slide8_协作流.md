## 🔄 协作流程

### 典型协作场景：PTO-GBA V2.0 项目

```
时间线: 2026-03-27

Step 1: 铲屎官派单
  → 铲屎官在 Discord 提出需求："研究 GBA 在 MoE 场景下的软件兜底率"

Step 2: 傅立叶建立课题
  → 白板创建 Task ID: 20260327_PTOGBA_V2
  → 填写 [Live Status] 进度条

Step 3: 傅立叶派发给多普勒
  → 调研指令：MoE 场景并发特征 + 真实并发规模 + 激励方案设计
  → 多普勒搜索论文、提取数据

Step 4: 多普勒交付调研
  → 填写 [Shared Specs]:
    • MoE 并发: 每迭代 262,144 同步点
    • 资源耗尽临界点: >1,536 任务触发软件兜底
    • Straggler 比例: 5-20%
    • 三种激励场景 (2000/5000/10000 任务)
  → 艾特傅立叶复核

Step 5: 傅立叶派发给我
  → 编码指令：基于规格实现 SystemC 仿真器
  → 我读取 [Shared Specs] 中的参数

Step 6: 我独立工作
  → 创建工作区: /root/shared_memory/hinton/20260327_PTOGBA_V2/
  → 启动进度条，推送到 Discord
  → 编码 + 自测

Step 7: 我交付白板
  → 填写 [Deliverables]:
    • 代码路径
    • 仿真结果: 软件兜底率 20.5%
  → 艾特傅立叶 Code Review

Step 8: 傅立叶复核结案
  → 验证结果符合预期
  → 生成结案报告
  → 归档到 `/root/shared_memory/fourier/20260327_PTOGBA_V2/结案报告.md`
```

### 关键约定

1. **唯一共享路径**: `/root/shared_memory/`
2. **白板是协作中心**: 所有异步通信通过白板
3. **各自目录隔离**:
   - 傅立叶: `/root/shared_memory/fourier/`
   - 多普勒: `/root/shared_memory/doppler/`

## 🚫 协作禁忌

| 禁忌 | 后果 | 正确做法 |
|------|------|---------|
| 直接改别人的区域 | 冲突/覆盖 | 只写自己负责的区域 |
| 代码放 /tmp/ | 别的猫读不到 | 必须放共享目录 |
| 跳过白板直接通信 | 信息不同步 | 所有任务通过白板 |
| 规格不清就开工 | 返工 | 先追问，再动手 |
